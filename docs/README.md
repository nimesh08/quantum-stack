# docs/

Two folders:

- **`specs/`** — the six specification documents (design blueprints, fixed
  inputs to the build). Treat as source-of-truth design; nothing in
  `specs/` is regenerated by the build.
- **`build/`** — phase build records (progress logs, usage guides,
  decisions). See `build/README.md`. This is where the agent writes its
  trail as it implements each phase.

The handoff brief itself (`specs/00_implementation_handoff_and_repo_strategy.docx`)
lives in `specs/` — read it first.
