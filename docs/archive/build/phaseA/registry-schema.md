# Spinor Registry — Canonical Schema

The registry is the data infrastructure that makes one
compiler serve a dozen-plus chips without a dozen-plus code
paths. Adding a new chip is **a YAML file, not a code change**
— this document is the contract for that YAML.

This file is the canonical schema. The machine-readable JSON
Schema lives at
[`spinor/registry/schema/spinor-registry.schema.json`](../../../spinor/registry/schema/spinor-registry.schema.json)
(landed in M4); this file explains every field in plain
English and gives one worked example per chip family.

## Chip entry — every field

```yaml
id: <string>                     # used in `target <id>`. Must match the file name.
provider: <ibm|aws|azure|rigetti|ionq|quantinuum|local>
qubits: <int>                    # physical qubit count
native_gates: [<gate>, ...]      # only gates decomposition may emit
coupling_map:                    # how the qubits are wired
  topology: <topology-name>      # name of a file in topologies/
  size: <int>                    # passed to parametric topologies
supports:
  mid_circuit_measure: <bool>    # relaxes W4
  feedforward: <none|limited|full>  # picks QIR Base vs Adaptive
  reset: <bool>                  # whether `reset` is a real op
calibration:
  source: <ibm_runtime_api|braket_api|azure_api|fixture>
  refresh: <nightly|daily|never|fixture>
  store: <path>                  # where the nightly job writes refreshed data
decomposition:
  one_qubit:
    recipe: euler_zyz
    rotation_gate: <native gate name>      # e.g. rz
    pi_2_gate:    <native gate name | null> # e.g. sx
  two_qubit:
    recipe: kak
    entangler: <native 2q gate name>       # e.g. ecr, ms, rzz
    entangler_count_max: 3                 # by KAK
pricing:                                  # used by resource estimator
  per_shot_usd: <float>
  per_minute_usd: <float | null>
notes: |
  Free-form. Anything a future contributor needs to know.
```

### Field-by-field semantics

- **`id`** — the device identifier the user puts in `target ...`.
  Must match the file's basename. The loader rejects mismatches.

- **`provider`** — selects which submission adapter handles this
  device in M10. Values are limited to the adapters we ship.

- **`qubits`** — number of physical qubits. The verifier uses it
  to bound qubit indices under the hardware contract.

- **`native_gates`** — the *only* gate names decomposition may
  emit. The verifier rejects a `target <id>` program that uses
  any other gate. The decomposer rejects a chip whose native
  set has no recipe.

- **`coupling_map.topology`** — must be the basename of a file
  in `topologies/`. That file is the wire list. Trapped-ion
  chips use `all_to_all` (a complete graph generated by `size`).
  Heavy-hex and other lattice topologies are stored as explicit
  edge lists.

- **`supports.mid_circuit_measure`** — when `true`, the
  verifier's W4 rule (no operations on a measured qubit until
  reset) is relaxed.

- **`supports.feedforward`** — `none` → emit QIR Base profile
  only; `limited` or `full` → may emit QIR Adaptive profile if
  the program needs it.

- **`supports.reset`** — when `false`, the decomposer rewrites
  `reset` to a measure-and-conditionally-flip sequence. When
  `true`, `reset` passes through as-is.

- **`calibration.source`** — selects the runtime adapter that
  fetches calibration. `fixture` means the registry ships with
  hard-coded data; used for IonQ Aria proto and CI.

- **`calibration.store`** — a path under
  `~/.cache/spinor/calibration/` (default), where the nightly
  job writes refreshed data and the compiler reads it on every
  invocation.

- **`decomposition.one_qubit.recipe`** — must be `euler_zyz` for
  Phase A.

- **`decomposition.one_qubit.rotation_gate`** — the chip's
  continuous Z-rotation (e.g. `rz` on IBM, `gpi`/`gpi2` plus
  `phase` virtual rotation on IonQ — see notes).

- **`decomposition.one_qubit.pi_2_gate`** — the chip's
  hardware-native π/2 rotation, if any (e.g. `sx` on IBM). Used
  by the ZYZ recipe to express `Ry(β)` as `sx · rz(β) · sxdg` or
  similar identities. May be null for chips with continuous
  rotations only.

- **`decomposition.two_qubit.entangler`** — the chip's native
  two-qubit gate. KAK uses up to three of these to express any
  two-qubit unitary.

- **`decomposition.two_qubit.entangler_count_max`** — must be
  `3` (the KAK theorem bound). Present so a CI sanity check
  rejects a registry that claims fewer.

- **`pricing.per_shot_usd`** — the per-shot price the resource
  estimator multiplies by the requested shot count.

- **`pricing.per_minute_usd`** — the per-machine-minute price
  for trapped-ion machines that bill on time. Null for chips
  that bill per shot.

- **`notes`** — anything else. The loader ignores it.

## Worked examples per chip family

### Superconducting / heavy-hex (IBM Heron r2)

```yaml
id: ibm_heron_r2
provider: ibm
qubits: 156
native_gates: [ecr, rz, sx, x]
coupling_map:
  topology: heavy_hex_156
  size: 156
supports:
  mid_circuit_measure: true
  feedforward: limited
  reset: true
calibration:
  source: ibm_runtime_api
  refresh: nightly
  store: ~/.cache/spinor/calibration/ibm_heron_r2.json
decomposition:
  one_qubit:
    recipe: euler_zyz
    rotation_gate: rz
    pi_2_gate: sx
  two_qubit:
    recipe: kak
    entangler: ecr
    entangler_count_max: 3
pricing:
  per_shot_usd: 0.00033
  per_minute_usd: null
notes: |
  Heron r2 = 156 qubits in a heavy-hex lattice. ECR is the
  native two-qubit; rz is virtual (zero-duration phase update
  in software).
```

### Trapped-ion all-to-all (IonQ Forte)

```yaml
id: ionq_forte
provider: aws
qubits: 36
native_gates: [gpi, gpi2, ms]
coupling_map:
  topology: all_to_all
  size: 36
supports:
  mid_circuit_measure: false
  feedforward: none
  reset: false
calibration:
  source: braket_api
  refresh: nightly
  store: ~/.cache/spinor/calibration/ionq_forte.json
decomposition:
  one_qubit:
    recipe: euler_zyz
    rotation_gate: gpi          # parameterised; see decomposer notes
    pi_2_gate: gpi2
  two_qubit:
    recipe: kak
    entangler: ms
    entangler_count_max: 3
pricing:
  per_shot_usd: 0.0125
  per_minute_usd: 0.30
notes: |
  All-to-all coupling; routing inserts no SWAPs. Forte is
  reachable through AWS Braket; native MS gate is the
  Mølmer-Sørensen interaction.
```

```yaml
decomposition:

    pi_2_gate: null
  two_qubit:
    recipe: kak

  program branches on measurement.
```

### The 4th chip — `ionq_aria_proto` (YAML-only demo)

The 4th chip is the proof of the "no code change" Definition
of Done. It is a fictional but plausible IonQ Aria-class
device, configured purely by YAML to demonstrate that
supporting a new chip requires no compiler changes. Lives
alongside the real chips in `registry/chips/ionq_aria_proto.yaml`
and runs through the same end-to-end test corpus.

## Versioning policy

The registry schema is versioned with the engine; a top-level
`schema_version: 1` is added in M4. Bumping the schema is a
decision-log entry.
